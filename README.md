# Action Influence Map 기반 강화학습 오목 AI

---

## 프로젝트 개요

기존 강화학습은 상태공간(State Space)이 커질수록 계산량이 기하급수적으로 증가하는 문제가 있다. 본 연구는 두 단계의 연구를 거쳐 **Action Influence Map** 기반 강화학습을 완성하였으며, 오목(Gomoku) 게임 환경에서 몬테카를로 강화학습에 적용해 성능과 학습 효율을 동시에 향상시키는 방법을 제안한다.

---

## 연구 흐름

```
[Traditional Influence Map]
        │  한계 인식 (계산량 증가, 영향력 상쇄 문제)
        ▼
[컨셉 1] 흑·백 돌 간 영향력 계산 기반 몬테카를로 오목 AI
        │  영향력 계산 방식 정형화 및 고도화
        ▼
[컨셉 2] Action Influence Map 기반 강화학습
```

---

## Traditional Influence Map

Traditional Influence Map은 지형에 존재하는 사물이나 에이전트들이 미치는 요소를 격자(grid) 위에 표현하여, 각 위치의 전략적·전술적 중요도를 결정하는 방법이다.

### 개념
- 지형을 격자 형태로 나누어 각 요소의 영향력 값을 주변으로 전파
- 전파된 값은 격자마다 누적되어 **증폭** 또는 **상쇄**
- 값이 상대적으로 큰 위치 → 전술적 주요 위치
- 값이 0에 가까운 위치 → 전선(Front Line)

### 한계

| 문제 | 설명 |
|---|---|
| **계산량 증가** | 영향을 미치는 요소가 많아질수록 계산량이 급증 |
| **영향력 상쇄 문제** | 서로 다른 부호의 값이 상쇄되면 각각의 원래 크기를 알 수 없게 됨 |
| **행동 공간 미해결** | 강화학습의 action space 문제를 직접 해결하는 방법이 없음 |

## 컨셉 1 — 흑·백 돌 간 영향력 계산 기반 몬테카를로 오목 AI

> *몬테카를로 방식의 높은 승률 + 오목의 넓은 상태공간 문제를 함께 해결하기 위한 초기 연구*

### 핵심 아이디어
틱택토(Tic-Tac-Toe)에서 몬테카를로 방식이 다른 알고리즘 대비 월등히 높은 승률을 보임을 확인하였다. 이를 오목에 확장 적용할 때 상태공간 폭발 문제가 발생하여, **돌 간 영향력(Influence)** 을 이용한 착수 제한으로 계산량을 줄이는 방법을 고안하였다.

### 환경 설정
- 보드 크기: **10×10** (81칸 유효)
- 흑 돌 = `1`, 백 돌 = `-1`, 빈 칸 = `0`
- 동일 크기의 **영향력 보드**를 별도로 관리

### 영향력 계산 방식

두 돌이 **인접(adjacent)** 할 때부터 영향력이 발생하며, 진행 방향으로 최대 3칸에 영향을 미친다.

| 칸 번호 (진행 방향 기준) | 상대적 영향력 크기 |
|:---:|:---:|
| 1번째 칸 (가장 가까운 칸) | `4x` |
| 2번째 칸 | `2x` |
| 3번째 칸 | `x` |

- 흑은 양의 값(+), 백은 음의 값(-)으로 영향력 부호를 구분
- **먼저 결정된 부호는 게임 종료까지 유지** → 상쇄 방지
- 상대방의 영향력이 자신의 돌에 의해 증폭될 수 있으며, 이는 상대의 착수 위치를 예측하는 데 활용됨

### 절벽 및 상대 돌 탐색 (Cliff & Enemy Search)

진행 방향 앞뒤 3칸 내에 **절벽(보드 끝)** 이나 **상대 돌**이 있으면 공격 가능성이 낮아진다. 이를 탐색하여 영향력 크기를 조정한다.

```
탐색 1) 장애물이 3번째 위치 → 인덱스 2 → 먼 쪽 최대 영향력 = x(h=x+2)
탐색 2) 장애물이 2번째 위치 → 인덱스 1 → 먼 쪽 최대 영향력 = x(h=x+1)
탐색 3) 장애물이 1번째 위치 → 인덱스 0 → 먼 쪽 영향력 모두 0
```

### 착수 제한 방법

인접한 돌의 쌍 개수 `k`를 구하고, 범위에 따라 상태가치를 계산할 위치 수 `n`을 결정한다.  
영향력 절대값이 큰 순서대로 상위 `n`개의 위치만 탐색하여 **계산량을 대폭 절감**한다.

## 컨셉 2 — Action Influence Map 기반 강화학습

> *컨셉 1의 영향력 개념을 정형화하여 Window 기반의 Action Influence Map으로 발전*  

### 핵심 아이디어
컨셉 1의 방식을 **Window 기반**으로 재정의하여 계산 방식을 일반화하였다. 영향력 값이 높은 상위 **10개의 행동만** 선택하여 강화학습의 계산량을 줄이고 성능을 향상시킨다.

### 환경 설정
- 보드 크기: **9×9**
- 흑 돌 = `1`, 백 돌 = `-1`, 빈 칸 = `0`
- 승리 조건: 동일한 색의 돌 **5개 연속** 착수

### Action Influence Map 생성 과정

```
① 현재 상태에서 승리 가능한 모든 Window(연속 5칸) 탐색
        │
② Window 내 빈 칸의 영향력 값 계산
   - 이번 착수 돌과 같은 색: (돌 개수 + 1)²
   - 이번 착수 돌과 다른 색: (돌 개수)²
        │
③ 흑 돌 기준 영향력 분포도 생성 (가로·세로·대각선 4방향 합산)
        │
④ 백 돌 기준 영향력 분포도 생성 (동일 방식)
        │
⑤ 두 분포도를 합산 → 최종 Action Influence Map 완성
        │
⑥ 상위 10개 영향력 값의 위치를 행동 후보로 선택
        │
⑦ Monte Carlo 강화학습 수행
```

### Traditional Influence Map과의 차이

| 항목 | Traditional | Action Influence Map (컨셉 1) | Action Influence Map (컨셉 2) |
|---|:---:|:---:|:---:|
| 영향력 계산 기준 | 거리 기반 전파 | 인접 돌 쌍 + 진행 방향 | Window(연속 5칸) 기반 |
| 영향력 상쇄 | 발생함 | 부호 고정으로 방지 | 부호 분리로 방지 |
| 행동 제한 방식 | 없음 | 인접 쌍 개수 k 기반 n개 | 상위 10개 고정 |
| 일반화 가능성 | 높음 | 오목 룰 의존 | 타 보드게임 확장 가능 |

---

## 실험 결과

실험 환경: 9×9 오목, 선공·후공 각 **100 에피소드**

### 행동 후보 수 비교

| 방법 | 평균 행동 후보 수 |
|---|:---:|
| **컨셉2 (Action Influence Map)** | **10개 고정** |
| Traditional RL (주변 8칸 제한) | 가변 |

### 계산 속도 비교

| 방법 | 선공 평균 | 후공 평균 | 전체 평균 |
|---|:---:|:---:|:---:|
| **컨셉 2** | 35.90s | 34.45s | **35.18s** |
| Traditional RL | 19.87s | 18.38s | 19.18s |

> ⚠️ 행동 후보 수는 줄었지만, 매 착수마다 Influence Map을 새로 생성하는 비용으로 인해 학습 시간은 기존 대비 약 **183%** 더 소요됨

### 승률 비교

| 시나리오 | 컨셉 2 승률 | Traditional RL 승률 | 차이 |
|---|:---:|:---:|:---:|
| 컨셉 2 **선공** | **79%** | 21% | **+58%** |
| 컨셉 2 **후공** | **53%** | 47% | **+6%** |

---

## 💡 결론

- 컨셉 1의 영향력 개념을 Window 기반으로 정형화하여 **계산 방식의 일반화** 달성
- 행동 후보를 **10개로 고정**함으로써 탐색 공간을 획기적으로 축소
- Traditional RL은 행동 공간이 커서 강제 제한이 필요한 반면, 제안 방법은 **최적 행동 선택을 자연스럽게 보장**
- 선공 기준 **79% 승률**로 기존 대비 58% 높은 성능 달성
- 향후 오셀로(Othello), 다이아몬드 게임 등 **다양한 보드게임**에 확장 적용 가능

---

## ⚠️ 한계 및 아쉬운 점

- Influence Map 생성 비용으로 인해 **학습 시간이 183% 증가**
- 후공 시 성능 향상폭이 상대적으로 작음 (선공 대비 +6%)
- Window 기반 계산 방식이 게임 룰에 의존하여, 타 게임 적용 시 **Window 재정의 필요**
